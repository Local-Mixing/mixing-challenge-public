At first, reversibility seems to limit the expressivity of reversible circuits. For example, how could we build a pseudorandom permutation, if the circuit could just be run in reverse?

Here's one idea. $$F,G$$ are two random reversible circuits:

![trapdoor perm](/assets/img/ckt/tdp.png){: width="90%"}

On input $$x$$, we compute $$F(x)$$, and then extract its lower $$n$$ bits (call this $$F(x)_n$$) with CNOT gates to output $$y$$. $$G(\cdot)$$ operates on the upper bits, hiding the full output of $$F(\cdot)$$. Overall, this circuit computes $$x\|0\mapsto G(F(x)) \| F(x)_n$$. Since $$G(\cdot)$$ is a random reversible circuit, $$G(F(\cdot))$$ should be a pseudorandom permutation independent of $$F(\cdot)$$. Furthermore, $$F(x)_n$$ may act as a compressing hash function (from $$(n+k)$$ to $$n$$ bits).

Observe that, in some sense, this circuit only allows us to run $$F(x)$$ in the forward direction. Since $$G(\cdot)$$ obscures the upper outputs of $$F(\cdot)$$, using $$y$$ as an input isn't particularly useful without being able to set the values of the upper $$k$$ bits.

Cite SZ25? Truncated permutation

We can get a slightly cleaner construction if we set $$G=F^{-1}$$; then $$r=x$$ and the overall functionality of the circuit is $$x\|0\mapsto x\| F(x)_n$$. In this case, the circuit is actually symmetric, $$C=C^{-1}$$.
